# つぎまでの時間が反応しない原因

## 症状

数字表示パネルで、残り時間は `16:55` のように残っているのに、`つぎまで` が `00:00` のままになることがありました。

## 原因

`つぎまで` は、現在の駅に保存されている `minutes` を使って計算しています。

```text
現在の駅.minutes → つぎまで
```

ところが、過去の版や旧データでは、時間が「現在の駅」ではなく「次の駅」側に保存されていることがありました。

```text
おうち.minutes = 0
きがえ.minutes = 5
```

この場合、本当は「おうち → きがえ」が5分でも、現在の駅である `おうち.minutes` が0なので、`つぎまで` が `00:00` になります。

一方、残り時間は別の合計計算を使っていたため、`16:55` のように残って見えていました。  
つまり、`残り時間` と `つぎまで` の計算で参照している時間データがずれていました。

## 修正

`おうち.minutes = 0` かつ `きがえ.minutes > 0` のような旧データを見つけた場合、時間を現在駅側へ移して扱います。

また、合計時間も `intervalMinutes()` を使って計算するようにし、`残り時間` と `つぎまで` の計算モデルをそろえました。

## 期待される表示

```text
残り時間 16:55
進み具合 1/5
つぎまで 05:00
```

のように、現在の区間時間が表示されます。
