# 「時計にあわせる」実装仕様と表示競合の修正

## 位置づけ

3モード化の第7段階として、「時計にあわせる」を現在時刻だけで動作する独立モードへ整理します。

併せて、第6段階の自動タイマーで発生していた「現在」「つぎまで」「予定」の表示のちらつきを修正します。

対象：

- `clock`：時計にあわせる
- `auto`：自動タイマーの表示競合修正

## 1. ちらつきの原因

従来の `app.js` は、起動時に登録した `render` を1秒ごとに実行します。

```javascript
setInterval(render, 1000)
```

一方、自動タイマーは専用表示を約500ミリ秒ごとに更新していました。

そのため同じDOM要素へ、次の2種類の値が交互に書き込まれていました。

```text
従来表示：おうち駅つうか中／あと5分／0/5
専用表示：おうち 1番目／あと04:35／1/5
```

## 2. 表示更新の一本化

`three-mode-auto-render.js` を3モード共通の表示コーディネーターとして使用します。

処理順：

```text
従来のrenderがDOMを更新
↓
checkStationEventsの最後で専用モードを確定
↓
ブラウザが画面を描画
```

自動タイマーまたは時計モードでは、従来表示が画面に出る前に専用表示へ確定します。

起動時に登録済みの旧 `render` タイマーなど、動的ラッパーを経由しない更新に備えて、`MutationObserver` でもDOM更新後・描画前に専用表示を確認します。

これにより、次の欄を1種類の表示で維持します。

- 現在
- つぎまで／つぎ
- 予定
- 上部の残り時間
- 現在の絵カード
- 次の予定
- 路線上の現在位置

## 3. 時計モードの基本原則

「時計にあわせる」では、端末の現在時刻だけを基準にします。

次の操作は使用しません。

- 出発
- 一時停止
- 再出発
- 最初の駅に戻る
- できた！

時間が過ぎたことを「できた」とは扱いません。

## 4. 4つの時刻状態

### 4.1 開始前

```text
はじまるまで　00:10
7:30から「おうち」です
```

大きいカードには最初の予定を「つぎの予定」として表示します。

### 4.2 予定中

開始時刻以上、終了時刻未満の場合です。

```text
この予定ののこり　02:15
いまは「きがえ」の時間です
```

大きいカードには「いまの予定」と表示します。

### 4.3 予定と予定の間

前の予定が終了し、次の予定がまだ始まっていない状態です。

```text
つぎまで　05:00
つぎは7:45から「あさごはん」です
```

大きいカードには次の予定を表示します。

### 4.4 全予定時刻の終了後

```text
終了　時間終了
この予定は終了しました。おつかれさまでした
```

「ぜんぶできました」「よくできました」は使用しません。

## 5. 進捗欄

時計モードでは次の名称に統一します。

```text
現在
つぎ
予定
```

表示例：

```text
現在　きがえ／07:38まで
つぎ　07:45から／あさごはん
予定　2/5
```

`2/5` は実施済み数ではなく、時刻上の位置です。

## 6. 路線表示

各駅は次の状態で表示します。

- 現在の予定
- 次の予定
- 時刻通過済み
- これから
- 時間終了

時計モードでは、従来の `done` クラスを使用しません。

車両位置は、実効開始時刻から実効終了時刻までの現在時刻の割合で表示します。

## 7. 実効開始時刻・終了時刻

共通時間計算では路線用の開始・終了位置を使用していますが、時計モードでは次の時刻も考慮します。

- 最初の予定の開始時刻
- 最後の予定の終了時刻

例：

```text
最初の予定　7:30〜7:35
最後の予定　8:00〜8:05
```

この場合の時計モード全体は7:30〜8:05です。7:31は最初の予定中、8:04は最後の予定中として表示します。

## 8. 深夜をまたぐ予定

次のような予定にも対応します。

```text
23:50〜23:55　開始
00:10〜00:20　休憩
00:40　　　　終了
```

0時台は翌日の時刻として1440分を加えて判定します。

## 9. 旧完了状態の分離

時計モード中は次の値を使用しません。

- `running`
- `timerStartedAt`
- `pausedElapsedMs`
- `pausedByLate`
- `doneUntilIndex`
- `clockDoneIndexes`
- `celebrated`

表示確定時にこれらを停止・初期化し、時計モードで紙吹雪や「できた」表示が出る経路を遮断します。

## 10. 編集画面

時計モードでは入力欄を次の名称に変更します。

```text
到着 → 開始時刻
出発 → 終了時刻
```

時計モードでは使用しない次の設定を隠します。

- ゆっくり運転の猶予
- 時間を過ぎたときの対応

別モードへ切り替えると再表示します。

- 最後の予定の終了時刻
- 時刻の24時間表記への変換

## 14. 第7段階の完了条件

- 自動タイマーの「現在」「つぎまで」「予定」がちらつかない
- 時計モードが現在時刻だけで動作する
- 時計モードに開始・停止操作が表示されない
- 時計モードに「できた！」が表示されない
- 開始前・予定中・予定間・終了後を区別できる
- 現在の絵カードと時刻表示が一致する
- 時計モードで「できた数」を表示しない
- 終了後に実施完了を断定しない
- 深夜をまたぐ予定に対応する
- 最初の開始時刻と最後の終了時刻を正しく使用する
- 時計モード専用の編集表記になる
- できた！タイマーと自動タイマーの動作を維持する
