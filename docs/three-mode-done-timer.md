# できた！タイマー安定化仕様

## 位置づけ

3モード化の第5段階として、「できた！タイマー」を他の2モードから独立させ、完了操作と時間表示を安定させます。

対象：

- `timer`：できた！タイマー

この段階では、自動タイマーの開始・自動進行は実装しません。

## 1. 進行の基本ルール

できた！タイマーでは、現在の予定が進む条件を「できた！」の操作に限定します。

```text
予定の時間が残っている
  ↓ できた！
次の予定へ進む

予定時間が0になった
  ↓ そのまま待つ
できた！
  ↓
次の予定へ進む
```

時間が0になっただけでは、現在の絵カードと完了位置を変更しません。

## 2. 次の予定は先頭から開始する

「できた！」を押した時刻に関係なく、次の予定はその予定の開始位置から始めます。

### 早めに終わった場合

予定の途中で「できた！」を押しても、次の予定の開始位置へ移動します。

### 遅れて終わった場合

予定時間を大きく超えてから「できた！」を押しても、超過時間を次の予定へ持ち越しません。

例：

```text
きがえ 5分
あさごはん 10分
```

「きがえ」を12分後に完了した場合でも、「あさごはん」は10分の先頭から始まります。

## 3. 一時停止中の完了

一時停止中に「できた！」を押した場合：

- 次の予定の開始位置へ移動する
- 一時停止状態を維持する
- 利用者が再出発するまで時間を進めない

「時間を過ぎたとき：できた！まで待つ」で停止した場合は、完了後に次の予定を再開します。

## 4. 最後の予定

最後の予定では、残り時間ではなく次を表示します。

```text
さいごの予定
できたらゴール
```

最後の「できた！」で次を行います。

- 完了数を全予定数にする
- タイマーを停止する
- ゴール表示を出す
- 完了音を鳴らす（音が有効な場合）
- 紙吹雪を表示する
- 「ぜんぶできました」と読み上げ領域へ通知する

## 5. 現在の予定に対応した残り時間

ヘッダーの時間表示を、全体の残り時間ではなく現在の予定に対応させます。

表示例：

```text
この予定ののこり
04:32
```

時間が0になった場合：

```text
この予定ののこり
おじかんになったよ
```

数字を見せない設定では、次のようなやさしい表現を使います。

- あと少し
- もうすぐ
- すすんでいます

## 6. 絵カードとの一致

現在の大きい絵カード、残り時間、「できた！」の対象、進捗数は、すべて `doneUntilIndex` から求めた同じ現在予定を参照します。

これにより、時間だけが次へ進み、絵カードが前の予定のままになる不一致を防ぎます。

## 7. 連続操作の防止

「できた！」を短時間に複数回押しても、1つの予定だけが完了するようにします。

- 完了処理中はボタンを一時的に無効化
- 450ミリ秒以内の再操作を無視
- 完了済みの場合は以後の操作を受け付けない

## 8. 追加モジュール

### 純粋な進行ルール

```text
three-mode-timer-core.js
```

主なAPI：

```javascript
activeIndex(schedule, doneUntilIndex)
taskTiming(schedule, doneUntilIndex, elapsedMs)
completeCurrent(schedule, runtime)
```

画面やブラウザ保存に依存しないため、自動テストができます。

### アプリとの接続

```text
three-mode-done-timer.js
```

担当する内容：

- 「できた！」操作の捕捉
- 連打防止
- 次の予定の開始位置への移動
- 現在予定の残り時間表示
- 最終完了処理
- 保存と再描画

## 9. 読み込み順

```text
three-mode-time.js
↓
three-mode-time-adapter.js
↓
three-mode-timer-core.js
↓
three-mode-done-timer.js
```

第五段階の読込に失敗した場合は、従来の「できた！」処理を継続します。

## 10. テスト

追加ファイル：

```text
tests/three-mode-done-timer.test.js
```

確認項目：

- 最初の予定が正しく選ばれる
- 現在予定の残り時間
- 時間超過の判定
- 早めの完了
- 時間超過後の完了
- 一時停止中の完了
- 「できた！」まで待った後の再開
- 最後の予定の完了
- 完了後の再操作を無視

実行方法：

```bash
node tests/three-mode-done-timer.test.js
```

## 11. 第5段階の完了条件

- 時間が0になっても自動的に次へ進まない
- 「できた！」1回につき1予定だけ進む
- 連打しても予定を飛び越さない
- 次の予定が先頭から始まる
- 超過時間を次の予定へ持ち越さない
- 一時停止中の状態を維持できる
- 現在の絵カードと残り時間が一致する
- 最後の「できた！」でゴールになる
- 完了後は「できた！」を操作できない
- 既存の時計にあわせるモードを変更しない

## 12. 第5段階では行わないこと

- 自動タイマーを開始可能にする
- 自動タイマーの一時停止・再開
- 時間経過による自動移動
- 自動タイマーの完了表示
- 時計にあわせるモードから「できた！」記録を分離する

これらは第6段階以降で実装します。
