# 3モード保存仕様 V21

## 位置づけ

3モード化の第2段階として、保存データを次の3種類のモードに対応させます。

- `timer`：できた！タイマー
- `auto`：自動タイマー
- `clock`：時計にあわせる

この段階では、自動タイマーの選択画面と自動進行処理はまだ実装しません。

## 保存キー

新しい保存キーは次のとおりです。

```text
oshitakuTrainNoPhotoStateV21
```

V20以前の保存キーは削除しません。V21がない場合は既存データを読み込み、V21形式として保存します。

## 追加する状態

```javascript
{
  schemaVersion: 21,
  mode: "timer" | "clock",
  requestedMode: "timer" | "auto" | "clock",
  autoRunning: false,
  autoStartedAt: null,
  autoPausedElapsedMs: 0
}
```

### `mode`

現在の画面と進行処理が実際に使用するモードです。

第2段階では自動タイマーの画面・進行処理が未実装のため、`requestedMode` が `auto` の場合も `mode` は安全な `timer` として開きます。

### `requestedMode`

利用者またはプリセットが希望している3種類のモードを保存します。

第3段階以降で自動タイマーの画面を追加したときに、この値をそのまま利用できます。

### 自動タイマー用状態

- `autoRunning`：自動タイマーが進行中か
- `autoStartedAt`：自動タイマーを開始した時刻
- `autoPausedElapsedMs`：一時停止までの経過時間

第2段階では初期化・検証・保存のみ行い、進行計算にはまだ使用しません。

## プリセット

各プリセットに次の項目を追加します。

```javascript
{
  defaultMode: "timer" | "auto" | "clock"
}
```

### 既存プリセット

モード情報を持たない既存プリセットは、互換性を優先して `timer` として移行します。

### 新しく保存するプリセット

保存時点で選択している進み方を `defaultMode` に保存します。

### プリセットを適用した場合

一覧、To Do、初回ガイドなど、どの経路からプリセットを開いても `defaultMode` を復元します。

`defaultMode: "auto"` のプリセットは、第2段階では `requestedMode: "auto"` を保持しながら、実際の画面は `timer` として安全に開きます。

## モード値の検証

保存時と読み込み時にモード値を検証します。

許可する値：

```text
timer
auto
clock
```

不正な値または空の値は `timer` に戻します。

旧名称や移行用の値は次のように変換します。

| 旧値・別名 | 変換後 |
|---|---|
| `manual` / `done` | `timer` |
| `automatic` / `schedule` | `auto` |
| `today` / `realtime` | `clock` |

## 旧データの扱い

1. V21があればV21を読み込む。
2. V21がなければ、既存のapp.jsがV20以前を読み込み・検証する。
3. 検証済みの状態に第2段階の項目を追加する。
4. V21として保存する。
5. V20以前のデータは削除しない。

これにより、既存の駅、プリセット、To Do、表示設定を維持します。

## 第2段階の完了条件

- 保存形式がV21になっている。
- `timer`、`auto`、`clock` を保存・検証できる。
- 自動タイマー用の状態を保存できる。
- プリセットに `defaultMode` が保存される。
- 既存プリセットは `timer` として安全に移行される。
- V20以前のデータが削除されない。
- 未実装の `auto` が既存の画面動作を壊さない。

## 第2段階では行わないこと

- 自動タイマーの選択ボタン追加
- 3モードのカード型選択画面
- 自動タイマーの時間計算
- 自動タイマーの出発・一時停止・再開
- 自動タイマーの自動的な駅移動
- モードごとの「できた！」ボタン表示切替

これらは第3段階以降で実装します。
