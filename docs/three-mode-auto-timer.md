# 自動タイマー実装仕様

## 位置づけ

3モード化の第6段階として、「自動タイマー」を実際に開始・一時停止・再開し、設定した所要時間に合わせて自動的に次の予定へ進めるようにします。

対象：

- `auto`：自動タイマー

## 1. 基本動作

自動タイマーでは、「出発」を押した時点から経過時間を計測します。

```text
出発
↓
1番目の予定
↓ 設定時間が終了
2番目の予定へ自動移動
↓
最後の時間が終了
↓
予定の時間が終了しました
```

利用者が「できた！」を押す必要はありません。

## 2. 使用する時間

できた！タイマーと同じ `intervalMin` を使用します。

例：

```text
おうち → 5分 → きがえ → 10分 → ごはん → 3分 → ゴール
```

内部の境界：

```text
0分 → 5分 → 15分 → 18分
```

経過時間が5分になった時点で「きがえ」、15分で「ごはん」、18分で終了へ移ります。

## 3. 開始

編集画面では次の操作を利用できます。

- この予定で始める
- プレビュー
- 出発
- 一時停止
- 最初の駅に戻る

「この予定で始める」を押すと、自動タイマーを開始して見るモードへ移ります。

「プレビュー」は時間を開始せず、見るモードの表示だけを確認します。

## 4. 一時停止と再開

一時停止時には、それまでの経過時間を `autoPausedElapsedMs` に保存します。

```javascript
{
  autoRunning: false,
  autoStartedAt: null,
  autoPausedElapsedMs: 540000
}
```

再出発すると、保存された経過時間から続きを開始します。

一時停止中は端末の時計が進んでも、自動タイマーの経過時間は増えません。

## 5. 自動移動

500ミリ秒ごとに現在の経過時間を確認します。

各予定の境界を越えた場合：

- 現在の絵カードを更新
- 次の予定を更新
- 車両位置を更新
- 残り時間を新しい予定に対応させる
- 進捗を `予定 2/5` の形式で更新
- 音または読み上げが有効なら新しい予定を通知

時間が進んだことを「できた」とは扱いません。

## 6. 「できた！」ボタン

自動タイマーでは「できた！」ボタンと、そのボタンを置く操作行を非表示にします。

```text
できた！タイマー：表示
自動タイマー：非表示
時計にあわせる：第7段階で整理
```

自動タイマーの進捗は、完了数ではなく現在の位置を表します。

## 7. 表示文

進行中：

```text
この予定ののこり
04:32
```

次の予定：

```text
つぎは「きがえ」です
```

進捗：

```text
現在　あさごはん 3番目
つぎまで　あと 05:24
予定　3/5
```

終了後：

```text
予定の時間が終了しました。
おつかれさまでした。
```

実施確認をしていないため、「ぜんぶできました」は使用しません。

## 8. 終了

最後の境界へ到達すると、次の処理を行います。

- `autoRunning` を `false` にする
- `autoStartedAt` を `null` にする
- 経過時間を合計時間に固定する
- 「予定の時間が終了しました」を表示する
- 完了音を1回だけ再生する
- 読み上げ領域へ終了を通知する
- 「もう一度出発」を利用可能にする

紙吹雪や「よくできました」は表示しません。

## 9. 完了後の再出発

完了後に「もう一度出発」または「もう一度始める」を押すと、経過時間を0へ戻して最初から開始します。

## 10. 保存と再読込

V21保存データの次の値を使用します。

```javascript
{
  requestedMode: "auto",
  autoRunning: true,
  autoStartedAt: 1710000000000,
  autoPausedElapsedMs: 300000,
  autoCompletedNotified: false
}
```

進行中にページを閉じて再度開いた場合：

- `autoStartedAt` から現在までの経過を復元
- 経過済みの予定位置へ自動的に戻る
- 合計時間を超えていれば終了状態へする

`requestedMode` が `auto` 以外の場合は、古い自動走行状態を停止・初期化します。

## 11. プリセットとの連携

`defaultMode: "auto"` のプリセットを適用した場合、自動タイマーが選択されます。

プリセットを切り替えた場合や駅の時間を変更した場合は、誤った途中位置から開始しないよう自動タイマーを最初へ戻します。

## 12. 追加ファイル

### 状態遷移の純粋ロジック

```text
three-mode-auto-core.js
```

主なAPI：

```javascript
elapsedMs(runtime, nowMs, totalMs)
snapshot(schedule, elapsedMs)
start(schedule, runtime, nowMs)
pause(schedule, runtime, nowMs)
reset()
finish(schedule)
```

### アプリ操作と表示

```text
three-mode-auto-timer.js
```

担当：

- 開始・一時停止・再開・リセット
- 時間境界による自動移動
- 自動終了
- 終了通知
- 操作ボタンの表示
- 「できた！」の非表示
- 駅の「通過済み／現在／これから」表示

### 再描画同期

```text
three-mode-auto-render.js
```

既存アプリが1秒ごとに再描画しても、自動タイマー専用の文言・進捗・ボタン状態を維持します。

## 13. 読み込み順

```text
three-mode-time.js
↓
three-mode-time-adapter.js
↓
three-mode-timer-core.js
↓
three-mode-done-timer.js
↓
three-mode-auto-core.js
↓
three-mode-auto-timer.js
↓
three-mode-auto-render.js
```

途中の追加ファイルを読み込めない場合は、読み込み済みの既存機能を継続します。

## 14. テスト

追加ファイル：

```text
tests/three-mode-auto-timer.test.js
```

確認項目：

- 初期位置
- 開始後の経過時間
- 境界時刻での予定切替
- 現在予定の残り時間
- 一時停止
- 停止中に時間が進まないこと
- 再開
- 保存状態からの経過時間復元
- 最終境界での終了
- 完了後の再出発

実行方法：

```bash
node tests/three-mode-auto-timer.test.js
```

## 15. 第6段階の完了条件

- 自動タイマーを開始できる
- 一時停止・再開できる
- 設定時間で次の予定へ自動移動する
- 現在の絵カードと残り時間が一致する
- 「できた！」が表示されない
- 進捗を「できた数」ではなく予定位置として表示する
- 最後の時間で自動停止する
- 「予定の時間が終了しました」と表示する
- 再読込後に進行状態を復元できる
- 完了後に最初から再出発できる
- できた！タイマーの動作を変更しない

## 16. 第6段階では行わないこと

- 時計にあわせるモードの操作・文言の最終整理
- 時計にあわせるモードの「できた！」非表示の完全分離
- 3モード全体を通したブラウザE2Eテスト
- 端末スリープ復帰時の専用メッセージ

これらは第7段階以降で実装します。
