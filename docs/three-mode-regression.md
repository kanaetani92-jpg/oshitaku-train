# 第9段階：3モード総合テスト・表示調整

## 位置づけ

第9段階では、第1〜8段階で実装した3モードを横断して確認し、表示の不一致、境界条件、再読み込み、画面回転、狭い画面でのはみ出しを調整します。

対象モード：

- できた！タイマー
- 自動タイマー
- 時計にあわせる

## 1. 総合確認の観点

### 端末と画面

- スマートフォン縦向き
- スマートフォン横向き
- タブレット
- PC
- 横タイムライン
- 縦タイムライン
- 自動タイムライン
- 高さの低い横向き画面

### 予定データ

- 1駅だけの予定
- 2駅の予定
- 標準的な5駅の予定
- 7駅以上の予定
- 長い駅名を含む予定
- 0時をまたぐ予定
- 旧保存データから移行した予定

### 操作状態

- 開始前
- 走行中／予定中
- 一時停止
- 予定と予定の間
- 最後の予定
- 完了／時間終了
- ページ再読み込み後
- ブラウザ履歴からの復帰
- アプリをバックグラウンドから戻した直後
- 画面回転後

## 2. 「数字を見せる」の統一

従来は「数字を見せる」をオフにしても、自動タイマーと時計モードの一部に時刻や秒数が残る場合がありました。

第9段階では、3モード共通で次のように整理します。

### 数字を表示する場合

```text
02:30
あと 02:30
2/5
07:45まで
07:50から
```

### 数字を表示しない場合

```text
あと少し
もうすぐ
すすんでいます
まっています
はじめ
進行中
もうすぐ最後
さいごの予定
できた！
時間終了
```

数字を消しても上部の時間欄そのものは隠しません。秒数の代わりに、現在の状態を示す言葉を表示します。

また、次の時刻表示も非表示にします。

- 路線上の駅の時刻
- 「これからすること」カードの時刻

## 3. できた！タイマー

### 数字表示時

進捗欄は、現在位置ではなく実際に押した「できた！」の数を表示します。

```text
できた　2/5
```

### 数字非表示時

現在位置を次の言葉で表示します。

```text
開始前
はじめ
進行中
もうすぐ最後
さいごの予定
できた！
```

最後の予定は、時間が0でも「できた！」を押すまで完了扱いにしません。

## 4. 自動タイマー

数字表示時は残り時間と位置を表示します。

```text
この予定ののこり　02:30
つぎまで　　　　　あと 02:30
予定　　　　　　　2/5
```

数字非表示時は次のように表示します。

```text
この予定ののこり　もうすぐ
つぎまで　　　　　もうすぐ
予定　　　　　　　進行中
```

最後まで進んだ場合だけ、次の表示へ切り替えます。

```text
時間終了
```

## 5. 時計にあわせる

時計モードでは、開始前・予定中・予定の間・終了後を維持します。

数字非表示時は実時刻を画面へ出さず、次のように表示します。

### 開始前

```text
まっています
つぎは「おうち」です
```

### 予定中

```text
すすんでいます
いまは「きがえ」の時間です
```

### 予定の間

```text
もうすぐ
つぎは「あさごはん」です
```

### 終了後

```text
時間終了
この予定は終了しました。おつかれさまでした
```

## 6. 1駅だけの予定

1駅だけの場合、完了前と完了後を明確に分けます。

### 完了前

```text
さいごの予定
できたらゴール
```

### 完了後

できた！タイマー：

```text
できた！
```

自動タイマー・時計にあわせる：

```text
時間終了
```

未完了の1駅予定を、位置だけを理由に完了表示へしません。

## 7. 再読み込み・アプリ復帰

次のイベントで、選択中モードの表示を再確定します。

- `pageshow`
- ウィンドウのフォーカス復帰
- `visibilitychange`
- `resize`
- `orientationchange`
- 「数字を見せる」の変更

自動タイマーの開始時刻、停止時間、時計モードの現在時刻を読み直し、表示を戻します。

## 8. 最終描画順

表示競合を防ぐため、読み込みと描画を次の順序へ固定します。

```text
3モードの時間計算
↓
できた！タイマー
↓
自動タイマー
↓
時計にあわせる
↓
第9段階のCSS
↓
第9段階の共通表示ルール
↓
第9段階の最終表示調整
↓
3モード最終描画コーディネーター
```

画面更新時は次の順です。

```text
従来のrender
↓
自動タイマーまたは時計モードの専用表示
↓
第9段階の数字・言葉表示
↓
ブラウザ描画
```

第9段階では独立したDOM監視を追加せず、既存の最終描画コーディネーターから呼び出します。これにより、複数の監視処理が互いに表示を書き戻すことを防ぎます。

## 9. 狭い画面の調整

第9段階専用CSSで、次を追加します。

- 時間を言葉で表示した場合の折り返し
- 進捗欄の文字サイズ調整
- 360px以下の画面への対応
- 高さ480px以下の横向き画面への対応
- タイムライン・カード・進捗欄の最大幅制限
- 長い駅名の折り返し
- 数字非表示時の駅時刻・カード時刻の非表示

既存の縦表示・横表示のレイアウト規則は変更しません。

## 10. 予定データの診断

表示時に次の状態を確認します。

- 駅が0件
- 駅が1件
- 合計時間が0以下
- 駅位置が数値でない
- 駅位置が逆順

問題を検出した場合は、画面を停止せず、安全な正規化済み表示を継続します。診断結果は開発者向けに`body`のデータ属性とコンソールへ記録します。

## 12. 回帰テスト

第9段階のみ：

```bash
node tests/three-mode-regression.test.js
```

3モード全体：

```bash
node tests/run-three-mode-tests.js
```

一括テストは次の順で実行します。

1. 共通時間計算
2. できた！タイマー
3. 自動タイマー
4. 時計にあわせる
5. 編集画面・プリセット
6. 第9段階の総合表示

途中で1件でも失敗すると、終了コード1で停止します。

## 13. 第9段階の完了条件

- 3モードで数字表示の方針が一致する
- 数字非表示時にも現在状態が理解できる
- タイマーの数字表示は実際の「できた数」を示す
- 1駅だけでも未完了と完了を区別する
- 最後の予定を早く完了扱いにしない
- 再読み込み後に選択中モードへ戻る
- アプリ復帰後に現在時刻と経過時間を再確認する
- 画面回転後に表示を再計算する
- 360px以下の画面で横にはみ出さない
- 横向きの低い画面で主要情報を維持する
- 複数の描画処理が交互に表示を書き換えない
- 旧データや不完全なデータで画面を停止しない
- Node.jsの第9段階回帰テストが正常終了する

## 14. 実機で確認する項目

コード上の回帰テストとは別に、公開後に次を実機確認します。

- iPhone Safariの縦向き・横向き
- Android Chromeの縦向き・横向き
- iPad Safari
- PCのChrome、SafariまたはEdge
- Vercel再デプロイ後のキャッシュ更新
- 音と読み上げの実際の再生
- スリープ復帰後の現在表示

実機でのみ確認できる項目は、第9段階のコード反映後も別途確認記録を残します。
