# 第8段階：編集画面・プリセット・To Doの3モード対応

## 位置づけ

第8段階では、編集画面、プリセット、To Doから作る新しい予定を、次の3モードへ対応させます。

- できた！タイマー
- 自動タイマー
- 時計にあわせる

第5〜7段階で実装した各モードの進行処理は変更せず、編集・保存・復元を接続します。

## 1. モード別の編集項目

### できた！タイマー

各予定について、次の予定へ進む目安時間を設定します。

```text
この予定に使う時間（分）
```

時間が終了しても自動では進まず、「できた！」を押すまで現在の予定を維持します。

### 自動タイマー

各予定を表示してから、自動で次へ移るまでの時間を設定します。

```text
この予定を表示する時間（分）
```

できた！タイマーとは別の値として保存します。

### 時計にあわせる

各予定について実際の時刻を設定します。

```text
開始時刻
終了時刻
```

端末の現在時刻が開始時刻以上、終了時刻未満の場合、その予定を「いまの予定」として表示します。

## 2. 最後の予定

できた！タイマーと自動タイマーでは、最後の予定に「次の予定までの時間」はありません。

そのため、既存仕様どおり最後の駅には分数入力欄を表示しません。

```text
🏁 ゴール
さいごにすることです。次の駅までの時間はありません。
```

時計にあわせるでは、最後の予定にも開始時刻と終了時刻を設定できます。

## 3. モード別の保存項目

駅データへ次の項目を追加します。

```javascript
{
  timerIntervalMin: 5,
  autoIntervalMin: 3,
  clockStart: "07:35",
  clockEnd: "07:38"
}
```

既存処理との互換性のため、次の項目も維持します。

```javascript
{
  intervalMin: 5,
  arrive: "07:35",
  depart: "07:38"
}
```

画面で選択中のモードに応じて、互換項目へ使用する値を反映します。

## 4. モードを切り替えた場合

モードを変更する前に、現在入力されている値を該当モード専用の項目へ保存します。

例：

```text
できた！タイマー　7分
自動タイマー　　　3分
時計にあわせる　　7:35〜7:38
```

できた！タイマーから自動タイマーへ切り替えると3分が表示され、再びできた！タイマーへ戻すと7分が復元されます。

時計モードの時刻も同様に保持されます。

## 5. 旧データの移行

モード別項目がない旧データでは、既存値を初期値として使用します。

```text
intervalMin → timerIntervalMin
intervalMin → autoIntervalMin
arrive      → clockStart
depart      → clockEnd
```

これにより、既存の予定を消さずに第8段階の形式へ移行します。

## 6. プリセット

プリセットには標準モードを保存します。

```javascript
{
  defaultMode: "auto"
}
```

### 保存時

現在選択しているモードと、そのモードを含む全時間データをプリセットへ保存します。

### 適用時

プリセットを適用すると、次を復元します。

- 予定名
- 乗り物
- 駅
- できた！タイマーの時間
- 自動タイマーの時間
- 時計モードの開始・終了時刻
- 標準モード

プリセット一覧には標準モード名を表示します。

## 7. To Doから作る新しい予定

To Doで「新しいプリセットを作る」を選んだ場合、次の3種類から標準モードを選択できます。

```text
できた！タイマー
自動タイマー
時計にあわせる
```

作成した新しいプリセットへ、その標準モードを保存します。

既存プリセットへ追加する場合は、新しいモードを上書きせず、そのプリセットの標準モードを維持します。そのため、モード選択欄は「新しいプリセットを作る」を選んだ場合だけ表示します。

## 8. 駅の追加・並べ替え・To Do挿入

次の操作でも、他モードの時間データを保持します。

- 駅を追加
- 上・下ボタンによる並べ替え
- ドラッグによる並べ替え
- 時計モードの「時刻で並べる」
- 駅を削除
- To Doを既存プリセットへ追加

時計モードで時刻をずらした場合は、`arrive` / `depart` と `clockStart` / `clockEnd` を同時に更新します。

## 9. 画面表示

編集画面上部に、選択中モードに応じた説明を表示します。

### できた！タイマー

```text
予定ごとに、「できた！」を押すまでの目安時間を設定します。
```

### 自動タイマー

```text
予定ごとに、自動で次の予定へ移るまでの表示時間を設定します。
```

### 時計にあわせる

```text
各予定を表示する開始時刻と終了時刻を設定します。
```

入力欄、説明、To Doのモード選択は、スマートフォンで横にはみ出さないよう幅を100%以内に制限します。

## 10. 既存画面の維持

第8段階では、次の既存構造を変更しません。

- To Do一覧カード
- ハンバーガーメニュー
- 使い方を見る
- スマートフォン下部の「予定／To Do」タブ
- できた！タイマーの進行
- 自動タイマーの進行
- 時計にあわせるの時刻判定

`index.html` には第8段階のスクリプト読み込みだけを追加します。

## 14. 第8段階の完了条件

- 3モードで編集項目が適切に切り替わる
- タイマーと自動タイマーの分数を別々に保持する
- 時計モードの開始・終了時刻を保持する
- モードを往復しても入力値が消えない
- 最後の予定に不要な分数入力を表示しない
- プリセットへ標準モードを保存する
- プリセット適用時に標準モードを復元する
- To Doから作る新しい予定へ標準モードを設定できる
- 既存プリセットへのTo Do追加では標準モードを変更しない
- 駅の追加・並べ替え・挿入で他モードの時間を失わない
- 既存のメニュー、To Do一覧、下部タブを維持する
