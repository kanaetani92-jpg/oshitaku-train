# 3モード選択画面仕様

## 位置づけ

3モード化の第3段階として、編集画面から次の3種類を選択できる画面を追加します。

- できた！タイマー
- 自動タイマー
- 時計にあわせる

この段階では、自動タイマーの選択と保存までを実装します。時間経過による自動進行処理は次の段階で追加します。

## 選択カード

各モードを、名称だけの小さいタブではなく、説明付きのカードとして縦に表示します。

### できた！タイマー

- アイコン：✅
- 説明：「できた！」を押して、自分のペースで進みます。
- 選択後の案内：「できた！」を押すと、次の予定へ進みます。

### 自動タイマー

- アイコン：▶️
- 説明：決めた時間が過ぎると、自動的に次へ進みます。
- 選択後の案内：選択内容を保存しました。自動進行の動作は次の段階で追加します。

### 時計にあわせる

- アイコン：🕐
- 説明：実際の時計に合わせて、現在の予定を表示します。
- 選択後の案内：端末の時計に合わせて、現在の予定を表示します。

## 選択状態

選択中のカードには次の表示を行います。

- 枠と背景を変更
- 右側にチェックを表示
- `aria-pressed="true"` を設定
- 読み上げ用の説明を付与

未選択カードは `aria-pressed="false"` とします。

## 自動タイマーの安全な扱い

第3段階では自動進行処理が未実装のため、自動タイマーを選択した場合は次のようにします。

- `requestedMode: "auto"` を保存する
- 画面は編集モードに維持する
- 「この予定で始める」を無効にする
- 「プレビュー」を無効にする
- タイマー操作欄を非表示にする
- 自動タイマー用の説明を駅編集欄に表示する

これにより、利用者が自動タイマーを通常のタイマーとして誤って開始することを防ぎます。

## プリセット表示

プリセット一覧に、保存されている標準モードを表示します。

表示例：

```text
朝の支度
朝の支度 / 🚃 / 5駅
できた！タイマー
```

モード表示は次の名称に統一します。

| 保存値 | 表示名称 |
|---|---|
| `timer` | できた！タイマー |
| `auto` | 自動タイマー |
| `clock` | 時計にあわせる |

## プリセットとの連携

- 新しいプリセットを保存すると、選択中のモードを `defaultMode` に保存する
- プリセットを適用すると、`defaultMode` に対応するカードを選択状態にする
- To Doや初回ガイドからプリセットを開いた場合も同様に復元する
- `auto` のプリセットは自動タイマー選択状態で編集画面を開く

## 画面幅への対応

- PC・タブレット・スマートフォンでカードを縦に並べる
- 小さい画面ではアイコン、文字、余白を縮小する
- モード名や説明が途中で切れないようにする
- キーボード操作とフォーカス表示に対応する
- 動きを減らす設定ではカードのアニメーションを停止する

## 追加ファイル

- `three-mode-ui.js`
- `three-mode-ui.css`

`three-mode-storage.js` から `three-mode-ui.js` を読み込みます。CSSは選択画面モジュールから読み込みます。

## 第3段階の完了条件

- 3種類のモードカードが表示される
- 各カードに名称と説明が表示される
- 選択中のカードが視覚的・アクセシブルに判別できる
- 選択したモードがV21へ保存される
- 再読み込み後も選択状態が復元される
- プリセット一覧に標準モードが表示される
- プリセット適用時に標準モードが復元される
- 自動タイマーを誤って開始できない
- 既存のタイマーおよび時計にあわせるの内部動作を変更しない

## 第3段階では行わないこと

- 自動タイマーの経過時間計算
- 自動タイマーの出発・一時停止・再開
- 時間経過による駅の自動移動
- 自動タイマーの終了判定
- 見るモードでの「できた！」ボタン切替
- モードごとの進捗欄変更

これらは第4段階以降で実装します。
