# おしたくトレイン

子どもが「いま何をするか」「つぎに何をするか」を絵カードと路線で確認できる予定支援Webアプリです。

## 3モード化

正式名称は次の3種類です。

- **できた！タイマー**：「できた！」を押して、自分のペースで進む
- **自動タイマー**：決めた時間が過ぎると、自動的に次へ進む
- **時計にあわせる**：実際の時計に合わせて、現在の予定を表示する

基本仕様は [`docs/three-mode-spec.md`](docs/three-mode-spec.md) を参照してください。

## 実装状況

### 第1段階：名称・仕様

完了しました。

- 3モードの正式名称を確定
- 進行基準、操作、進捗および終了表現を整理
- 「今日」に依存しない表現へ変更

### 第2段階：保存データ

完了しました。

- 保存形式をV21へ更新
- `timer`、`auto`、`clock` の3種類を保存・検証できる基盤を追加
- プリセットに `defaultMode` を追加
- 自動タイマー用の進行状態を保存できる項目を追加
- V20以前のデータを残したままV21へ移行
- 未実装の `auto` は希望値を保持しつつ、現在の画面では安全な `timer` として開く

保存仕様の詳細は [`docs/three-mode-storage-v21.md`](docs/three-mode-storage-v21.md) を参照してください。

### 第3段階：3モード選択画面

完了しました。

- 3種類を説明付きカードとして表示
- 選択中カードへチェック、枠、背景を表示
- `aria-pressed` と読み上げ用説明を追加
- 選択したモードをV21へ保存
- プリセット一覧へ標準モード名を表示
- プリセット適用時に選択モードを復元
- 自動タイマー選択時は開始・プレビューを無効化
- スマートフォンを含む狭い画面へ対応

選択画面の詳細は [`docs/three-mode-selection-ui.md`](docs/three-mode-selection-ui.md) を参照してください。

### 第4段階：共通時間計算

完了しました。

- `timer` と `auto` を所要時間型として共通化
- `clock` を実時刻型として分離
- 駅間時間、合計時間、現在位置を共通形式で計算
- 24時をまたぐ予定へ対応
- 自動タイマー用の経過時間計算APIを追加
- 共通計算と従来計算を起動時に比較
- 不一致・例外時は従来計算へ戻る安全策を追加
- Node.jsで実行できる自動テストを追加

時間計算の詳細は [`docs/three-mode-time-engine.md`](docs/three-mode-time-engine.md) を参照してください。

### 第5段階：できた！タイマーの安定化

完了しました。

- 時間が0になっても「できた！」まで現在の予定を維持
- 「できた！」1回につき1予定だけ進行
- 連打による予定の飛び越しを防止
- 早く終わった場合も次の予定を先頭から開始
- 遅れた時間を次の予定へ持ち越さない
- 一時停止中の完了では停止状態を維持
- 現在の絵カードと、その予定の残り時間を同期
- 最後の「できた！」でタイマー停止・ゴール・完了通知
- 完了後の再操作を無効化

できた！タイマーの詳細は [`docs/three-mode-done-timer.md`](docs/three-mode-done-timer.md) を参照してください。

## 現在の画面

編集画面では次の3モードを選択できます。

- できた！タイマー
- 自動タイマー
- 時計にあわせる

**できた！タイマー**は、開始、一時停止、再開、「できた！」による進行、最後のゴールまで利用できます。

**自動タイマー**は現在、選択・保存・プリセットへの設定・時間計算の準備まで利用できます。開始、一時停止、時間経過による自動進行は第6段階以降で実装します。
